• WasherCss

Washer Css is a BEM, clean, simple, easy, lightweight, customizable, themable css framework.

It's slightly inspired by InuitCSS

Structure

There are 4 principal folders inside washer :

  1. settings : defines some constants inside the project. you can redefine basic elements inside a settings/_specific.scss file
  2. helpers : basic classes that you can include into your components and that you can use inside your html templates
  3. components : components classes (BEM style)
  4. base : basic elements styling : can include some helpers

Settings

There is a _defaults.scss file here that you can copy inside your project for further refinement. You can change the constants (for color customization for example).

Helpers

This is the helpers of the project, some basic classes you can add to your components, to your basic elements (in a balanced way), and that you can use directly inside your markup.

Components

This folder is a BEM-style components list. It's quite simple to integrate. To explain BEM (block element modifier) in a minute :

Block is an block that you define. For example a nav. So inside your markup you'll have a div with the class "nav"

Then you have an element : an element inside a block (let's take a layout item), we'll call it .layout__item. If it's a link inside the layout, the name is .layout__link (.{block}__{element})

And a modifier : a class we can add to change a block (layout--tiny) or an element (layout__item--big). Here the "big" modifier is an example that doesn't exist in Washer.

Keep in mind that nothing is cascading in the BEM model. We try to do css as BUILDING BLOCKS for html that are not coupled to older css.

Look at the layout component for an example of a great component
